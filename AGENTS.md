# Repository Rules

These rules apply to every Codex task in this repository.

Task-specific instructions override this file.

## Workflow

- Never commit directly to `main`.
- Work on a feature branch.
- Open a Pull Request for every change.

## Implementation

- Implement only the requested task.
- Prefer the simplest solution that satisfies the request.
- Do not redesign the architecture unless explicitly requested.
- Do not change unrelated functionality.
- Never make assumptions.
- If anything is unclear, stop and ask for clarification.

## Review

- Focus on correctness before style.
- Report only meaningful issues.
- Do not suggest changes outside the requested task.

## Communication

- Respond in Japanese.
